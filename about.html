What is Private Set Intersection?

A further look into how it works, the benefits, and the research done.

Introduction to Private Set Intersection

Private Set Intersection (PSI) is a secure two-party computation (2PC) problem that allows for two parties to compute the intersection between both sets of data, while the elements that are not in the intersection remain private to the other party. There are many different approaches to this problem, however, for this project, we will be focusing on the Diffie-Hellman based PSI approach.

How does it actually work?

To illustrate, this is a basic process of how the Diffie-Hellman based Private Set Intersection works:

1. Initial Setup:
a. Two parties, known as Alice and Bob, agree on a DH group description, including a generator g and an order q.
b. They also have hash functions (e.g., H) to ensure the original set data stay hidden.

2. Protocol Execution:
The protocol consists of four rounds:

a. Alice:
I. Picks a secret random exponent a from the range [0,𝑞)
II. Hashes all items in her input set 𝑋
III. Raises the hashed set to the power of 𝑎 and sends the results to Bob. (𝑋 )

b. Bob:
I. Picks a secret random exponent b from the same range.
II. Follows the same procedure as Alice, with his input set 𝑌
III. Raise the hashed set to the power of 𝑏 and send the results to Alice. (𝑌 )

c. Alice:
I. Receives Bob’s message and raises it to the power of 𝑎. (𝑌 )
II. Send the result back to Bob.

d. Bob:
I. Receives Alice’s message and raises it to the power of 𝑏. (𝑋 )
II. Send the result back to Alice.

The intersected elements are now revealed, as both parties can compare the common elements in 𝑋 and 𝑌 .
As Alice knows the original elements in 𝑋, and Bob knows the original elements in 𝑌, both parties will know the original intersected elements, while remaining oblivious to the identity of the other elements.

Benefits Of Private Set Intersection

PSI allows privacy of the information to be preserved, while allowing for a meaningful exchange of information between two parties. No additional information aside from the intersected elements is disclosed,
ensuring both parties’ data remain confidential. This can be useful in multiple ways, and this is explored in the two examples below:
1. Voting - PSI allows the user to check if a voter is accidentally registered in 2 different states by checking if any registered voters intersect in the list of voters between the two states.
Doing this through the use of PSI maintains the privacy and identity of the other voters to the other state.

Healthcare - PSI allows for the user to compare his/her genes against the hospital’s database to compute the risk of a particular disease without revealing her genes. On the other hand,
the hospital also does not wish to reveal its discoveries to the user as the hospital has spent time and money researching the genes. As this data is private to both the user and the hospital,
PSI allows for the user to only discover the risk of the disease without revealing additional information.

Challenges Of Implementing Private Set Intersection

What are some challenges of implementing PSI in our application?

1. Asymmetric Gene Set Size

The specialised hospital is expected to have a significantly larger set of gene set data as compared to the patient. This would lead to an asymmetric gene set size between the two parties, which may significantly increase the computation time. Certain protocols need to be implemented to ensure the computation process remains efficient. To address this issue, we may look to implement cuckoo hashing can be used to ensure a constant time lookup of the data, ensuring that the application remains efficient.

2. Ensuring Privacy of Gene Sets

A collection of individual gene data is known and stored in a database. If hashing alone is used in the process of PSI, malicious attackers may use dictionary attacks to compromise the privacy of the patient or the hospital. To address this issue, homomorphic encryption can be used to ensure the privacy of both parties. Homomorphic encryption allows for computation of the data while it remains encrypted, allowing both parties to determine the intersected sets without revealing additional information.
